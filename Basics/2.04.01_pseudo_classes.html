CSS Pseudo Classes

Selecting elements based on UI state

Pseudo classes are used to define the special state of an element. For example, it can be used to style an element when a user mouses over it, or when a link is visited.

Pseudo classes are used in CSS selectors to style elements based on their state or position in the document tree. They are preceded by a colon (:) and can be used to target elements that are not easily selectable through regular selectors.

Here are the explanations for each type:

1. Elemental Pseudo Classes

Elemental pseudo classes are used to select elements based on their type or position in the document tree. Some commonly used elemental pseudo classes include:

  • :defined, matches any element that is defined with a custom element definition.
  • :heading, matches any element that is a heading element (h1, h2, h3, h4, h5, h6).

2. Elemental Display State Pseudo Classes

Elemental display state pseudo classes are used to select elements based on their display state. Some commonly used elemental display state pseudo classes include:

  • :open, matches any element that is open (e.g., a
    element).
  • :popover-open, matches any element that is open as a popover, or currently on the showing state.
  • :model, Matches an element that is in a state in which it excludes all interaction with elements outside it until the interaction has been dismissed.
  • :fullscreen, matches any element that is in fullscreen mode.

3. Input Pseudo Classes

Input pseudo classes are used to select form elements based on their state. Some commonly used input pseudo classes include:

  • :enabled, matches any form element that is enabled.
  • :disabled, matches any form element that is disabled.
  • :read-only, Represents any element that cannot be changed by the user.
  • :read-write, Represents any element that can be changed by the user.
  • :user-valid, Represents any element that has user input that is valid.
  • :user-invalid, Represents any element that has user input that is invalid.
  • :required, Represents any element that has the `required` attribute specified.
  • :optional, Represents any element that does not have the `required` attribute specified.

4. Linguistic Pseudo Classes

Linguistic pseudo classes are used to select elements based on their language or text direction. Some commonly used linguistic pseudo classes include:

  • :lang, matches any element that has a specific language attribute.
  • :dir, matches any element that has a specific text direction attribute.

5. Location Pseudo Classes

Location pseudo classes are used to select elements based on their location in the document tree. Some commonly used location pseudo classes include:

  • :any-link, matches any element that is a link.
  • :link, matches any element that is an unvisited link.
  • :visited, matches any element that is a visited link.
  • :local-link, matches any element that is a link to the same origin.
  • :target, matches any element that is the target of a fragment identifier in the URL.
  • :scope, matches any element that is the root of the current document or the root of a shadow tree.

6. Time-dimentional Pseudo Classes

Time-dimentional pseudo classes are used to select elements based on their temporal state. Some commonly used time-dimentional pseudo classes include:

  • :current, matches any element that is the current element in a time-based context.
  • :past, matches any element that is in the past in a time-based context.
  • :future, matches any element that is in the future in a time-based context.

7. Tree-Structural Pseudo Classes

Tree-structural pseudo classes are used to select elements based on their position in the document tree. Some commonly used tree-structural pseudo classes include:

  • :first-child, matches any element that is the first child of its parent.
  • :last-child, matches any element that is the last child of its parent.
  • :nth-child(n), matches any element that is the nth child of its parent.
  • :nth-last-child(n), matches any element that is the nth child of its parent, counting from the end.
  • :only-child, matches any element that is the only child of its parent.
  • :only-of-type, matches any element that is the only child of its parent with the same element type.

8. Shadow-Structural Pseudo Classes

Shadow-structural pseudo classes are used to select elements based on their position in the shadow tree. Some commonly used shadow-structural pseudo classes include:

  • :host, matches the host element of a shadow tree.
  • :host(), matches the host element of a shadow tree when a specific condition is met.
  • :host-context(), matches the host element of a shadow tree when a specific condition is met.
  • :has-slotted, matches any element that has slotted content.

9. User-action Pseudo Classes

User-action pseudo classes are used to select elements based on user interactions. Some commonly used user-action pseudo classes include:

  • :hover, matches any element that is being hovered over.
  • :active, matches any element that is currently being activated.
  • :focus, matches any element that has focus.
  • :focus-within, matches any element that has focus or contains an element that has focus.
  • :focus-visible, matches any element that has focus and the user agent determines that the element should receive focus visibility.
  • :target-current, matches any element that is the current target of the document.

10. Functional Pseudo Classes

Functional pseudo classes are used to select elements based on specific conditions. Some commonly used functional pseudo classes include:

  • :not(), matches any element that does not match a specific selector.
  • :is(), matches any element that matches a specific selector.
  • :where(), matches any element that matches a specific selector, but with lower specificity.
  • :has(), matches any element that has a specific descendant.

11. Custom State Pseudo Classes

Custom state pseudo classes are used to select elements based on custom states defined by the user. Some commonly used custom state pseudo classes include:

  • :state(), matches any element that has a specific custom state.

12. Page Pseudo Classes

Page pseudo classes are used to select elements based on their position in a paged media context. Some commonly used page pseudo classes include:

  • :first, matches the first page in a paged media context.
  • :left, matches any page that is positioned on the left side in a paged media context.
  • :right, matches any page that is positioned on the right side in a paged media context.
  • :blank, matches any page that is blank in a paged media context.

13. View Transition Pseudo Classes

View transition pseudo classes are used to select elements based on their state during a view transition. Some commonly used view transition pseudo classes include:

  • :active-view-transition, matches any element that is currently undergoing a view transition.
  • :active-view-transition-type, matches any element that is currently undergoing a view transition.
